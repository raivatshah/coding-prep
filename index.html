Problem-Solving Framework

A systematic approach for every coding problem. Internalize this flow.

Step 1: Understand

Before anything, make sure you truly understand the problem.

  • Restate the problem in your own words
  • What are the inputs and outputs?
  • What are the constraints? (n ≤ 10^5 means O(n log n) or better)
  • Can input be empty? Negative? Have duplicates?
  • Is input sorted? Can I modify it?
  • Walk through the given example by hand
Constraint-to-complexity mapping:
n ≤ 10 → O(n!) or O(2^n)  |  n ≤ 20 → O(2^n)  |  n ≤ 500 → O(n^3)
n ≤ 10^4 → O(n^2)  |  n ≤ 10^6 → O(n log n)  |  n ≤ 10^8 → O(n)

Step 2: Pattern Recognition

Match the problem to a known pattern. Ask yourself these questions:

If the problem involves...Consider...
Sorted array, find pair/tripletTwo Pointers
Contiguous subarray/substringSliding Window
Search in sorted data, minimize/maximize answerBinary Search
Next greater/smaller element, matching bracketsStack / Monotonic Stack
Top K, K-th largest/smallest, streamingHeap
Connected components, shortest path, levelsGraph BFS/DFS
All combinations/permutations/subsetsBacktracking
Overlapping subproblems + optimal substructureDynamic Programming
Dependency orderingTopological Sort
Dynamic connectivity, groupingUnion-Find
Prefix-based string searchTrie
Overlapping intervalsSort + sweep
Frequency counting, deduplicationHash Map / Set

Step 3: Implement

  • Start with the function signature — clarify return type
  • Handle base cases first — empty input, single element
  • Write the core algorithm — narrate each block
  • Use descriptive names — left, right not i, j for two pointers
  • Keep it simple — don't over-engineer; correctness > cleverness

Step 4: Verify & Optimize

  • Trace through your code with the given example, line by line
  • Test with edge cases: empty, single element, all same, large input
  • State time and space complexity
  • If asked to optimize, look for: redundant work, better data structures, space reduction

collections — Your Best Friend

from collections import defaultdict, Counter, deque, OrderedDict

# Counter — frequency counting in one line
c = Counter("aabbbcc")
c.most_common(2)        # [('b', 3), ('a', 2)]
c['z']                  # 0 (no KeyError!)
c1 - c2                 # subtract counts (keeps positive only)

# defaultdict — no more key checks

# deque — O(1) append/pop from both ends
q = deque([1, 2, 3])
q.appendleft(0)         # [0, 1, 2, 3]
q.popleft()             # 0, O(1)!  (list.pop(0) is O(n))
q.rotate(1)             # rotate right

# OrderedDict — LRU Cache building block
od = OrderedDict()
od.move_to_end(key)     # move to end (most recent)
od.popitem(last=False)  # pop oldest (FIFO)

bisect — Binary Search Built-in

▶ 
import bisect

sorted_arr = [1, 3, 5, 7, 9]

# bisect_left: first position >= target (lower bound)
bisect.bisect_left(sorted_arr, 5)    # 2

# bisect_right: first position > target (upper bound)
bisect.bisect_right(sorted_arr, 5)   # 3

# Insert while maintaining sort order
bisect.insort(sorted_arr, 6)         # [1, 3, 5, 6, 7, 9]

# Count occurrences in sorted array
def count(arr, target):
    return bisect.bisect_right(arr, target) - bisect.bisect_left(arr, target)

# LIS in O(n log n) uses bisect_left!
def lis(nums):
    tails = []
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)

        result[idx] = val
    stack.append(i)

Trees

Hierarchical structure. Binary trees (at most 2 children) dominate interviews. Key property: most tree problems are naturally recursive.

OperationBST (balanced)BST (worst)Notes

# Binary tree node (given in most problems)
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

# --- Three DFS traversals (recursive) ---
def preorder(node):             # Root → Left → Right
    if not node: return
    visit(node.val)
    preorder(node.left)
    preorder(node.right)

def inorder(node):              # Left → Root → Right (sorted for BST!)
    if not node: return
    inorder(node.left)
    visit(node.val)
    inorder(node.right)

def postorder(node):            # Left → Right → Root
    if not node: return
    postorder(node.left)
    postorder(node.right)
    visit(node.val)

# --- BFS (level-order) ---
from collections import deque

def level_order(root):
    if not root: return []
    result = []
    queue = deque([root])
    while queue:
        level = []
        for _ in range(len(queue)):     # process one full level
            node = queue.popleft()
            level.append(node.val)
            if node.left:  queue.append(node.left)
            if node.right: queue.append(node.right)
        result.append(level)
    return result

# --- Recursive thinking pattern ---
# Most tree problems follow: solve for children, combine at parent
def max_depth(root):
    if not root: return 0
    return 1 + max(max_depth(root.left), max_depth(root.right))
💡
BST key property: For every node, all values in the left subtree < node.val < all values in the right subtree. Inorder traversal of a BST yields elements in sorted order.

Recursion space: O(h) where h = height. Balanced tree: h = log n. Skewed tree: h = n. Always mention this in interviews.
Associated algorithms: DFS (preorder, inorder, postorder), BFS (level-order), BST search/insert/validate, LCA, diameter, path sums, serialize/deserialize. Tree DP = DFS that returns tuples.

A heap is a complete binary tree stored as an array where the parent is always smaller (min-heap) or larger (max-heap) than its children. Python's heapq module implements a min-heap only.

heapq.heappush(heap, 1)
heapq.heappush(heap, 2)

# Heapify existing list — O(n), NOT O(n log n)
nums = [5, 3, 8, 1, 2]
heapq.heapify(nums)             # nums is now a min-heap in-place

# Top-K pattern: min-heap of size k
def top_k(nums, k):
    heap = nums[:k]
    heapq.heapify(heap)
    for num in nums[k:]:
        if num > heap[0]:
            heapq.heapreplace(heap, num)  # pop smallest, push new
    return heap

# Tuple comparison for priority with tiebreaker
heapq.heappush(heap, (priority, tiebreaker, item))
When to use a heap: Top-K elements, K-th largest/smallest, merge K sorted lists, running median (two heaps), Dijkstra's shortest path, task scheduling by priority. If you see "K" and "largest/smallest/closest" in the same sentence, think heap.

Tries (Prefix Trees)

A tree where each node represents a character. Paths from root to nodes spell prefixes. Each operation is O(m) where m = word length — independent of how many words are stored.

OperationTimeNotes
Insert wordO(m)m = word length
Search wordO(m)Check is_end flag at last node
Search prefixO(m)Just walk the tree, no is_end check
SpaceO(total chars)Shared prefixes save space
class TrieNode:
    def __init__(self):
        self.children = {}      # char → TrieNode
        self.is_end = False     # marks end of a complete word

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word):     # exact match
        node = self._walk(word)
        return node is not None and node.is_end

    def starts_with(self, prefix):
        return self._walk(prefix) is not None

    def _walk(self, s):
        node = self.root
        for ch in s:
            if ch not in node.children:
                return None
            node = node.children[ch]
        return node
When to use a trie: Autocomplete, spell checker, word search in a grid (backtracking + trie), counting words with a given prefix, IP routing (prefix matching). If a problem involves searching many strings by prefix, a trie beats sorting + binary search.

Two Pointers

Two indices moving through data to reduce O(n²) to O(n). Works best on sorted arrays.

Pattern  When to Use

  • Sorted array — find pair that sums to target
  • Remove duplicates in-place
  • Container with most water, trapping rain water
  • Palindrome checking
  • Partitioning (Dutch national flag)

Template: Converging Pointers

▶ 
def two_sum_sorted(arr, target):
    """Find pair in sorted array that sums to target."""
    left, right = 0, len(arr) - 1
    while left < right:
        curr = arr[left] + arr[right]
        if curr == target:
            return [left, right]
        elif curr < target:
            left += 1      # need bigger sum
        else:
            right -= 1     # need smaller sum
    return [-1, -1]
# Time: O(n), Space: O(1)

            continue  # skip duplicates
        left, right = i + 1, len(nums) - 1
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            if total == 0:
                result.append([nums[i], nums[left], nums[right]])
                while left < right and nums[left] == nums[left+1]:
                    left += 1  # skip duplicates
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                left += 1
                right -= 1
            elif total < 0:
                left += 1
            else:
                right -= 1
    return result
# Time: O(n²), Space: O(1) excluding output

Template: Variable-Size Window

▶ 
def longest_k_distinct(s, k):
    """Longest substring with at most k distinct characters."""
    window = {}      # char → count
    left = 0
    best = 0

    for right in range(len(s)):
        # EXPAND: add right char to window
        window[s[right]] = window.get(s[right], 0) + 1

        # SHRINK: while window violates constraint
        while len(window) > k:
            window[s[left]] -= 1
            if window[s[left]] == 0:
                del window[s[left]]
            left += 1

        # UPDATE: window is valid, record best
        best = max(best, right - left + 1)

    return best
# Time: O(n), Space: O(k)
💡
The template: (1) Expand right, (2) Shrink left while invalid, (3) Update answer. This 3-step pattern solves most sliding window problems.

Template: Fixed-Size Window

def max_sum_subarray(arr, k):
    """Maximum sum of any subarray of size k."""
    if len(arr) < k:
        return 0
    window_sum = sum(arr[:k])
    best = window_sum

    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]  # slide: add new, remove old
        best = max(best, window_sum)

    return best
# Time: O(n), Space: O(1)

def next_greater_element(nums):
    """For each element, find the next greater element to its right."""
    n = len(nums)
    result = [-1] * n
    stack = []          # stores INDICES, not values

    for i in range(n):

        stack.append(i)

def diameter_of_tree(root):
    ans = 0
    def dfs(node):
        nonlocal ans
        if not node:
            return 0
        left = dfs(node.left)
        right = dfs(node.right)
        ans = max(ans, left + right)    # path through this node

Two Heaps: Running Median

class MedianFinder:
    def __init__(self):
        self.small = []     # max-heap (negate values!) — lower half
        self.large = []     # min-heap — upper half

    def addNum(self, num):
        # Always add to max-heap first
        heapq.heappush(self.small, -num)
        # Balance: move max of small to large
        heapq.heappush(self.large, -heapq.heappop(self.small))
        # Keep small >= large in size
        if len(self.large) > len(self.small):
            heapq.heappush(self.small, -heapq.heappop(self.large))

    def findMedian(self):
        if len(self.small) > len(self.large):
            return -self.small[0]
        return (-self.small[0] + self.large[0]) / 2

def permutations(nums):
    result = []
    def backtrack(path, used):
        if len(path) == len(nums):
            result.append(list(path))
            return
        for i in range(len(nums)):
            if used[i]:
                continue
            used[i] = True
            path.append(nums[i])
            backtrack(path, used)
            path.pop()
            used[i] = False
    backtrack([], [False] * len(nums))
    return result

def combination_sum(candidates, target):
    result = []

        if remaining == 0:
            result.append(list(path))
            return

            if candidates[i] > remaining:
                break                       # prune (requires sorted input)
            path.append(candidates[i])
            backtrack(i, path, remaining - candidates[i])   # i, not i+1 (reuse!)
            path.pop()
    candidates.sort()
    backtrack(0, [], target)
    return result

from collections import deque

# Standard BFS
def bfs(graph, start):
    visited = {start}
    queue = deque([start])
    while queue:
        node = queue.popleft()
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)   # mark BEFORE enqueueing!
                queue.append(neighbor)

# Shortest path in unweighted graph
def shortest_path(graph, start, end):
    queue = deque([(start, 0)])
    visited = {start}
    while queue:
        node, dist = queue.popleft()
        if node == end:
            return dist
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, dist + 1))
    return -1       # unreachable

# Multi-source BFS (e.g., Rotting Oranges LC 994)
def multi_source_bfs(grid):
    if not grid or not grid[0]:
        return 0
    rows, cols = len(grid), len(grid[0])
    queue = deque()
    fresh = 0

    # Initialize: add ALL sources at once
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2:         # rotten orange
                queue.append((r, c, 0))
            elif grid[r][c] == 1:
                fresh += 1

    time = 0
    while queue:
        r, c, t = queue.popleft()
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            nr, nc = r + dr, c + dc
            if 0 <= nr < rows and 0 <= nc < cols and grid[nr][nc] == 1:
                grid[nr][nc] = 2        # mark visited
                fresh -= 1
                time = t + 1
                queue.append((nr, nc, t + 1))

    return time if fresh == 0 else -1
⚡

Use for: course scheduling, build order, dependency resolution. Only works on DAGs.

# Kahn's Algorithm (BFS-based) — preferred in interviews
# Also detects cycles: if result length < n, there's a cycle
from collections import deque, defaultdict

def topo_sort(n, prerequisites):
    """Course Schedule II (LC 210)"""
    graph = defaultdict(list)
    in_degree = [0] * n

    for course, prereq in prerequisites:
        graph[prereq].append(course)
        in_degree[course] += 1

    # Start with nodes that have no prerequisites
    queue = deque([i for i in range(n) if in_degree[i] == 0])
    order = []

    while queue:
        node = queue.popleft()
        order.append(node)
        for neighbor in graph[node]:
            in_degree[neighbor] -= 1
            if in_degree[neighbor] == 0:
                queue.append(neighbor)

    # If we processed all nodes, no cycle
    return order if len(order) == n else []
# Time: O(V + E), Space: O(V + E)
Topological sort key idea: repeatedly remove nodes with 0 in-degree. If you can't remove all nodes, there's a cycle. 
# DFS-based topological sort
def topo_sort_dfs(n, edges):
    graph = defaultdict(list)
    for u, v in edges:
        graph[u].append(v)

    visited = set()
    result = []

    def dfs(node):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                dfs(neighbor)
        result.append(node)     # add AFTER processing all descendants

    for i in range(n):
        if i not in visited:
            dfs(i)

    return result[::-1]         # reverse post-order

Dijkstra's Algorithm

Use for: shortest path in weighted graph with non-negative edges

import heapq
from collections import defaultdict

def dijkstra(graph, start, n):
    """Returns shortest distance from start to all nodes."""
    dist = [float('inf')] * n
    dist[start] = 0
    heap = [(0, start)]         # (distance, node)

    while heap:
        d, u = heapq.heappop(heap)
        if d > dist[u]:
            continue            # skip stale entries (IMPORTANT!)
        for v, w in graph[u]:   # (neighbor, weight)
            new_dist = dist[u] + w
            if new_dist < dist[v]:
                dist[v] = new_dist
                heapq.heappush(heap, (new_dist, v))

    return dist
# Time: O((V + E) log V), Space: O(V + E)

# Network Delay Time (LC 743) — direct application
def network_delay(times, n, k):
    graph = defaultdict(list)
    for u, v, w in times:
        graph[u].append((v, w))

    dist = dijkstra(graph, k, n + 1)    # 1-indexed
    max_dist = max(dist[1:])
    return max_dist if max_dist < float('inf') else -1
⚡
The stale entry check (if d > dist[u]: continue) is critical for correctness and performance. Without it, you process nodes multiple times.
💡
When to use which shortest path algorithm:
• Unweighted graph: BFS (O(V+E))
• Non-negative weights: Dijkstra (O((V+E) log V))
• Negative weights: Bellman-Ford (O(VE))
• All-pairs: Floyd-Warshall (O(V³))
• Weights are 0 or 1: 0-1 BFS with deque (O(V+E))

Cycle Detection

# Directed graph: 3-color DFS (white=unvisited, gray=in-progress, black=done)
def has_cycle_directed(graph, n):
    WHITE, GRAY, BLACK = 0, 1, 2
    color = [WHITE] * n

    def dfs(node):
        color[node] = GRAY          # currently exploring
        for neighbor in graph[node]:
            if color[neighbor] == GRAY:
                return True         # back edge = cycle!
            if color[neighbor] == WHITE and dfs(neighbor):
                return True
        color[node] = BLACK         # done exploring
        return False

    return any(color[i] == WHITE and dfs(i) for i in range(n))

# Undirected graph: DFS with parent tracking
def has_cycle_undirected(graph, n):
    visited = set()

    def dfs(node, parent):
        visited.add(node)
        for neighbor in graph[node]:
            if neighbor not in visited:
                if dfs(neighbor, node):
                    return True
            elif neighbor != parent:    # visited and not parent = cycle
                return True
        return False

    return any(i not in visited and dfs(i, -1) for i in range(n))

1D Dynamic Programming

Single variable state: dp[i]

# Climbing Stairs (LC 70) — the hello world of DP
# dp[i] = number of ways to reach step i
# dp[i] = dp[i-1] + dp[i-2]
def climb_stairs(n):
    if n <= 2:
        return n
    prev2, prev1 = 1, 2
    for i in range(3, n + 1):
        prev2, prev1 = prev1, prev2 + prev1
    return prev1
# Space optimized: only need last 2 values

# House Robber (LC 198) — can't take adjacent elements
# dp[i] = max money from houses[0..i]
# dp[i] = max(dp[i-1], dp[i-2] + nums[i])
def rob(nums):
    prev2, prev1 = 0, 0
    for num in nums:
        prev2, prev1 = prev1, max(prev1, prev2 + num)
    return prev1

# Maximum Subarray (LC 53) — Kadane's Algorithm
# dp[i] = max sum ending at index i
# dp[i] = max(nums[i], dp[i-1] + nums[i])
def max_subarray(nums):
    if not nums:
        return 0
    max_sum = curr = nums[0]
    for num in nums[1:]:
        curr = max(num, curr + num)    # extend or start fresh
        max_sum = max(max_sum, curr)
    return max_sum

# Word Break (LC 139)
# dp[i] = can s[0:i] be segmented?
def word_break(s, wordDict):
    words = set(wordDict)
    dp = [False] * (len(s) + 1)
    dp[0] = True
    for i in range(1, len(s) + 1):
        for j in range(i):
            if dp[j] and s[j:i] in words:
                dp[i] = True
                break
    return dp[-1]

# Decode Ways (LC 91)
# dp[i] = ways to decode s[0:i]
def num_decodings(s):
    if not s or s[0] == '0':
        return 0
    prev2, prev1 = 1, 1
    for i in range(1, len(s)):
        curr = 0
        if s[i] != '0':
            curr += prev1       # single digit
        two = int(s[i-1:i+1])
        if 10 <= two <= 26:
            curr += prev2       # two digits
        prev2, prev1 = prev1, curr
    return prev1
Space optimization tip: If dp[i] only depends on dp[i-1] and dp[i-2], you don't need the full array — just keep two variables (prev1, prev2).

2D Dynamic Programming

Two variable state: dp[i][j]

# Unique Paths (LC 62)
# dp[i][j] = number of paths to reach (i, j)
# dp[i][j] = dp[i-1][j] + dp[i][j-1]
def unique_paths(m, n):
    dp = [[1] * n for _ in range(m)]
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i-1][j] + dp[i][j-1]
    return dp[-1][-1]

# Space optimized — single row
def unique_paths_opt(m, n):
    row = [1] * n
    for i in range(1, m):
        for j in range(1, n):
            row[j] += row[j-1]
    return row[-1]

# Minimum Path Sum (LC 64)
def min_path_sum(grid):
    if not grid or not grid[0]:
        return 0
    m, n = len(grid), len(grid[0])
    for i in range(m):
        for j in range(n):
            if i == 0 and j == 0: continue
            elif i == 0: grid[i][j] += grid[i][j-1]
            elif j == 0: grid[i][j] += grid[i-1][j]
            else: grid[i][j] += min(grid[i-1][j], grid[i][j-1])
    return grid[-1][-1]

Knapsack Problems

Three variants: 0/1 (each item once), unbounded (unlimited), and boolean (subset sum).

# 0/1 Knapsack — each item can be used AT MOST ONCE
# dp[w] = max value achievable with capacity w
def knapsack_01(weights, values, capacity):
    dp = [0] * (capacity + 1)
    for i in range(len(weights)):
        for w in range(capacity, weights[i] - 1, -1):   # REVERSE order!
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[capacity]

# Unbounded Knapsack — each item can be used UNLIMITED times
# Coin Change (LC 322) — minimum coins
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for a in range(coin, amount + 1):               # FORWARD order!
            dp[a] = min(dp[a], dp[a - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

# Coin Change II (LC 518) — count combinations
def coin_change_2(coins, amount):
    dp = [0] * (amount + 1)
    dp[0] = 1
    for coin in coins:              # outer loop = coins (combinations, not permutations)
        for a in range(coin, amount + 1):
            dp[a] += dp[a - coin]
    return dp[amount]

# Partition Equal Subset Sum (LC 416) — boolean knapsack
def can_partition(nums):
    total = sum(nums)
    if total % 2:
        return False
    target = total // 2
    dp = [False] * (target + 1)
    dp[0] = True
    for num in nums:
        for t in range(target, num - 1, -1):    # reverse for 0/1
            dp[t] = dp[t] or dp[t - num]
    return dp[target]
🚨
Critical distinction:
0/1 knapsack: iterate capacity in REVERSE (ensures each item used once)
Unbounded knapsack: iterate capacity FORWARD (allows reuse)
Getting this wrong is the #1 knapsack bug!

LCS, Edit Distance & String DP

# Longest Common Subsequence (LC 1143)
# dp[i][j] = LCS of text1[0..i-1] and text2[0..j-1]
def lcs(text1, text2):
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i-1] == text2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1        # chars match
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])  # skip one

    return dp[m][n]

# Edit Distance (LC 72)
# dp[i][j] = min operations to convert word1[0..i-1] to word2[0..j-1]
def edit_distance(word1, word2):
    m, n = len(word1), len(word2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(m + 1): dp[i][0] = i    # delete all
    for j in range(n + 1): dp[0][j] = j    # insert all

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if word1[i-1] == word2[j-1]:
                dp[i][j] = dp[i-1][j-1]            # no operation needed
            else:
                dp[i][j] = 1 + min(
                    dp[i-1][j],       # delete
                    dp[i][j-1],       # insert
                    dp[i-1][j-1]      # replace
                )
    return dp[m][n]

# Longest Palindromic Subsequence (LC 516)
# LPS(s) = LCS(s, reverse(s))
def longest_palindrome_subseq(s):
    return lcs(s, s[::-1])
String DP pattern: Almost always 2D where dp[i][j] considers prefixes of both strings. When chars match, come from diagonal. When they don't, take best of skipping either character.

Longest Increasing Subsequence (LIS)

# O(n²) DP — easy to understand and implement
def lis_dp(nums):
    if not nums:
        return 0
    n = len(nums)
    dp = [1] * n        # dp[i] = LIS ending at index i
    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

# O(n log n) — Patience Sorting with bisect
import bisect
def lis_fast(nums):
    if not nums:
        return 0
    tails = []          # tails[i] = smallest tail of LIS of length i+1
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)       # extend LIS
        else:
            tails[pos] = num        # replace with smaller value
    return len(tails)

# Example walkthrough:
# nums = [10, 9, 2, 5, 3, 7, 101, 18]
# tails evolution:
#   [10] → [9] → [2] → [2,5] → [2,3] → [2,3,7] → [2,3,7,101] → [2,3,7,18]
# LIS length = 4
💡
O(n log n) LIS trick: tails is NOT the actual LIS — it's just tracking the smallest possible tail for each length. The length of tails gives the LIS length.

State Machine DP

Model problems with distinct states and transitions between them.

# Best Time to Buy/Sell Stock with Cooldown (LC 309)
# States: held (holding stock), sold (just sold), rest (idle)
def max_profit_cooldown(prices):
    held = -prices[0]       # bought on day 0
    sold = 0                # impossible to have sold
    rest = 0                # no stock, idle

    for price in prices[1:]:
        new_held = max(held, rest - price)     # keep holding OR buy from rest
        new_sold = held + price                # sell what we're holding
        new_rest = max(rest, sold)             # stay idle OR transition from sold
        held, sold, rest = new_held, new_sold, new_rest

    return max(sold, rest)  # can't end in "held" state

# Best Time to Buy/Sell Stock with at most K transactions (LC 188)
def max_profit_k(k, prices):
    if not prices:
        return 0
    n = len(prices)

    # Optimization: if k >= n//2, it's unlimited transactions
    if k >= n // 2:
        return sum(max(0, prices[i+1] - prices[i]) for i in range(n-1))

    # dp[t][0] = max profit after t transactions, NOT holding
    # dp[t][1] = max profit after t transactions, HOLDING
    dp = [[0, float('-inf')] for _ in range(k + 1)]

    for price in prices:
        for t in range(k, 0, -1):
            dp[t][0] = max(dp[t][0], dp[t][1] + price)    # sell
            dp[t][1] = max(dp[t][1], dp[t-1][0] - price)  # buy (uses 1 transaction)

    return dp[k][0]
State machine DP recipe: (1) Identify the distinct states, (2) Define transitions between states, (3) Track the optimal value for each state at each step.

Bitmask DP

Use a bitmask to represent which items have been used/visited. Works when n ≤ 20.

# Traveling Salesman Problem (TSP)
# State: (current_city, bitmask_of_visited_cities)
from functools import lru_cache

def tsp(dist, n):
    ALL_VISITED = (1 << n) - 1

    @lru_cache(maxsize=None)
    def dp(mask, pos):
        if mask == ALL_VISITED:
            return dist[pos][0]         # return to start

        result = float('inf')
        for city in range(n):
            if mask & (1 << city):      # already visited
                continue
            new_mask = mask | (1 << city)
            result = min(result, dist[pos][city] + dp(new_mask, city))
        return result

    return dp(1, 0)     # start at city 0, mark it visited
# Time: O(2^n * n^2), Space: O(2^n * n)

# Partition to K Equal Sum Subsets (LC 698)
def can_partition_k(nums, k):
    total = sum(nums)
    if total % k:
        return False
    target = total // k
    nums.sort(reverse=True)     # optimization: try large numbers first
    n = len(nums)

    @lru_cache(maxsize=None)
    def dp(mask, curr_sum):
        if mask == (1 << n) - 1:
            return True
        for i in range(n):
            if mask & (1 << i):
                continue
            new_sum = curr_sum + nums[i]
            if new_sum > target:
                continue
            new_mask = mask | (1 << i)
            if new_sum == target:
                if dp(new_mask, 0):
                    return True
            else:
                if dp(new_mask, new_sum):
                    return True
        return False

    return dp(0, 0)
💡
Bitmask operations cheat sheet:
Check if bit i is set: mask & (1 << i)
Set bit i: mask | (1 << i)
Clear bit i: mask & ~(1 << i)
Count set bits: bin(mask).count('1')
All n bits set: (1 << n) - 1

Interval DP

Solve for all intervals [i, j], building from smaller intervals to larger ones.

# Burst Balloons (LC 312) — THE classic interval DP
# dp[i][j] = max coins from bursting all balloons between i and j (exclusive)
def max_coins(nums):
    nums = [1] + nums + [1]     # add boundary balloons
    n = len(nums)
    dp = [[0] * n for _ in range(n)]

    # Iterate by interval length
    for length in range(2, n):
        for left in range(0, n - length):
            right = left + length
            # Try every balloon k as the LAST one to burst in (left, right)
            for k in range(left + 1, right):
                dp[left][right] = max(
                    dp[left][right],
                    dp[left][k] + dp[k][right] + nums[left] * nums[k] * nums[right]
                )
    return dp[0][n - 1]

# Matrix Chain Multiplication pattern
# dp[i][j] = min cost to multiply matrices i through j
# Try every split point k: dp[i][j] = min(dp[i][k] + dp[k+1][j] + cost(i,k,j))
Interval DP key insight: Iterate by increasing interval length. For each interval [i, j], try every possible split point k. Think about what's the LAST operation in the interval.

Tree DP

Combine DP with tree traversal — each node's state depends on its children.

# House Robber III (LC 337) — rob houses in a binary tree
def rob_tree(root):
    def dfs(node):
        if not node:
            return (0, 0)       # (rob_this_node, skip_this_node)

        left = dfs(node.left)
        right = dfs(node.right)

        rob = node.val + left[1] + right[1]    # rob this, must skip children
        skip = max(left) + max(right)           # skip this, children can be robbed or not

        return (rob, skip)

    return max(dfs(root))

# Diameter of Binary Tree (LC 543)
def diameter(root):
    ans = 0
    def dfs(node):
        nonlocal ans
        if not node:
            return 0
        left = dfs(node.left)      # height of left subtree
        right = dfs(node.right)    # height of right subtree
        ans = max(ans, left + right)   # path through this node
        return 1 + max(left, right)    # return height to parent
    dfs(root)
    return ans
✓
Tree DP pattern: Return a tuple from DFS representing different states (e.g., include/exclude this node). The parent combines children's results to compute its own state.

Union-Find (Disjoint Set Union)

Dynamic connectivity — efficiently merge groups and check if elements are connected.

Template with Path Compression + Union by Rank

class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.components = n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py:
            return False            # already connected
        # Union by rank — attach shorter tree under taller tree
        if self.rank[px] < self.rank[py]:
            px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]:
            self.rank[px] += 1
        self.components -= 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)

# Time: O(α(n)) ≈ O(1) per operation (inverse Ackermann)

When to use

  • Connected components that merge dynamically
  • Detect cycles in undirected graph (if union returns False, edge creates cycle)
  • Kruskal's MST (sort edges, union greedily)
  • Accounts Merge (LC 721), Number of Provinces (LC 547)
  • Graph Valid Tree (LC 261): n-1 edges + all connected = tree

  • All identical elements
  • Already sorted (ascending and descending)
  • Negative numbers and zeros
  • Duplicates
  • Very large values (overflow? Python handles this, but think about it)

Linked Lists

  • Empty list (null head)
  • Single node
  • Operations on the head (deletion, insertion)
  • Operations on the tail
  • Cycle in the list

Trees

  • Empty tree (null root)
  • Single node
  • Skewed tree (all left or all right — like a linked list)
  • Perfect/complete binary tree
  • Negative values in nodes

Graphs

  • Empty graph / single node
  • Disconnected components
  • Cycles
  • Self-loops
  • Dense vs sparse
  • Negative edge weights (Dijkstra won't work!)

Numbers

  • Zero
  • Negative numbers
  • Integer overflow (use int(-7/2) for truncation toward zero)
  • Division by zero
  • Power of 2 boundaries

Common Mistakes

Avoid these and you're already ahead of most candidates.

Before Coding

  • ✗ Jumping straight into code without clarifying the problem
  • ✗ Not asking about constraints (input size tells you the required complexity!)
  • ✗ Not discussing approach before implementing
  • ✗ Skipping brute force mention (even if you know the optimal)

During Coding

  • ✗ Writing code in silence
  • ✗ Over-engineering or writing clever one-liners
  • ✗ Using single-letter variables everywhere
  • ✗ Going silent when stuck (verbalize!)
  • ✗ Ignoring interviewer hints

After Coding

  • ✗ Not tracing through with an example
  • ✗ Not analyzing time/space complexity unprompted
  • ✗ Not considering edge cases
  • ✗ Not mentioning possible optimizations

Python-Specific Gotchas

  • ✗ [[0]*n]*m — all rows share the same reference! Use [[0]*n for _ in range(m)]
  • ✗ list.pop(0) is O(n) — use deque.popleft() for O(1)
  • ✗ -7 // 2 == -4 in Python (floors toward -inf), not -3. Use int(-7/2) for truncation
  • ✗ Default mutable arguments: def f(x=[]) — the list is shared across calls
  • ✗ Forgetting sys.setrecursionlimit() for deep recursion
